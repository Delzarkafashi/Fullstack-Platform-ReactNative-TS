# Fullstack Platform React Native TypeScript

## Projektbeskrivning
Detta projekt är en fullstack webbplattform byggd med **React Native och TypeScript** i frontend samt en separat backend med databas.

Målet är att bygga en modern, tillgänglig och skalbar webbplats liknande kommunala informationssidor där **allt innehåll styrs från databasen**.

Projektet byggs steg för steg med tydlig planering och sprintar. Frontend byggs först som ett visuellt skal med komponenter och placeholder innehåll. Därefter kopplas backend och databas in och innehållet fylls på dynamiskt.

TypeScript används från start för tydligare struktur, bättre skalbarhet och säkrare kod när projektet växer.

## Syfte och mål
- Bygga en komplett fullstack webbplattform från grunden  
- Använda TypeScript för tydlig och skalbar kod  
- Skapa ett återanvändbart komponentbibliotek  
- Separera innehåll från kod genom databasstyrt innehåll  
- Ha tydlig struktur, bra UX och god tillgänglighet  
- Arbeta enligt en tydlig roadmap med sprintar

## Sprintplan

### Sprint 1 Grund och riktning
I denna sprint bestämmer vi vad projektet ska vara.  
Vi definierar mål, målgrupp, vilka sidor som ska finnas och hur allt ska hänga ihop.  
Fokus ligger på struktur och planering, inget byggande.

### Sprint 2 UI skal
Här bygger vi hela webbplatsen visuellt.  
Navigation, layout, sidor och grundstruktur sätts upp.  
Allt innehåll består av placeholder text och bilder men sidan ska kännas färdig.

### Sprint 3 Moduler och komponenter
Vi skapar alla återanvändbara byggblock.  
Exempel är sök, hero sektioner, kort, listor och informationsblock.  
Komponenterna används på sidorna men utan riktig data.

### Sprint 4 Data och koppling
Nu kopplas backend och databas in.  
Allt innehåll flyttas från frontend koden till databasen.  
Sidor och komponenter hämtar sitt innehåll via API.

### Sprint 5 Admin och innehåll
Vi skapar funktioner för att hantera innehåll.  
Skapa och redigera sidor, ändra texter, byta bilder och publicera innehåll.

### Sprint 6 UX och kvalitet
Finslipning av hela plattformen.  
Tillgänglighet, prestanda, tydlighet, tester och förbättrad användarupplevelse.
